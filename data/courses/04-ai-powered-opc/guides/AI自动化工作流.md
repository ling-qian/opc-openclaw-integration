# AI 自动化工作流

> 用 Zapier/Make 实现一人公司的自动化，让重复工作交给机器人。

---

## 一、自动化基础

### 什么是自动化

```
触发条件 → 自动执行 → 产出结果

示例：
有人私信咨询 → 自动发送欢迎消息 → 记录到客户表
```

### 常用工具对比

| 工具 | 特点 | 价格 | 适合 |
|------|------|------|------|
| Zapier | 最流行，集成最多 | 免费100任务/月 | 海外工具 |
| Make | 功能更强，可视化 | 免费1000操作/月 | 复杂流程 |
| 飞书多维表格 | 国内生态，免费 | 免费 | 国内用户 |
| 腾讯轻联 | 国内版Zapier | 免费基础版 | 国内用户 |

## 二、10个自动化场景

### 场景1：新客户自动建档

**触发**：收到新的私信/咨询
**动作**：自动在Notion/表格中创建客户记录

```
流程：
收到小红书私信 
→ 解析客户信息（昵称/需求）
→ 在Notion客户表中新增一行
→ 发送通知给你
```

**Zapier设置**：
1. Trigger：Gmail/微信（有新邮件/消息）
2. Action：Notion（创建数据库项）
3. Action：Slack/飞书（发送通知）

### 场景2：内容发布提醒

**触发**：内容日历中的日期到达
**动作**：自动提醒你发布内容

```
流程：
Notion内容日历 → 到达发布日期
→ 发送飞书/微信提醒
→ 附上内容标题和平台
```

### 场景3：成交自动记录

**触发**：收到付款通知
**动作**：自动记录到财务表

```
流程：
微信/支付宝收款通知
→ 解析金额和付款人
→ 在财务表中新增收入记录
→ 发送成交喜报到社群
```

### 场景4：客户跟进提醒

**触发**：客户表中的"下次跟进日期"到达
**动作**：自动提醒你跟进

```
流程：
Notion客户表 → 到达跟进日期
→ 发送提醒（包含客户名称和上次沟通内容）
→ 标记为"已提醒"
```

### 场景5：数据自动汇总

**触发**：每天/每周固定时间
**动作**：自动汇总数据并生成报表

```
流程：
每天22:00
→ 从各平台抓取今日数据（粉丝/浏览/互动）
→ 汇总到数据表
→ 生成今日简报
→ 发送到飞书/微信
```

### 场景6：自动回复常见问题

**触发**：收到包含关键词的消息
**动作**：自动发送预设回复

```
流程：
收到消息包含"价格/多少钱"
→ 自动回复价格表
→ 同时通知你有新咨询
```

### 场景7：内容素材收集

**触发**：看到好的内容/灵感
**动作**：自动保存到素材库

```
流程：
微信发送"收藏：[内容]"
→ 自动解析内容
→ 保存到Notion素材库
→ 分类打标签
```

### 场景8：订单自动交付

**触发**：收到付款
**动作**：自动发送产品/课程

```
流程：
收到付款
→ 确认金额
→ 自动发送课程链接/模板文件
→ 发送使用说明
→ 记录到订单表
```

### 场景9：竞品监控

**触发**：竞品发布新内容
**动作**：自动通知你

```
流程：
每天09:00
→ 检查竞品账号（RSS/API）
→ 发现新内容
→ 发送通知（附标题和链接）
```

### 场景10：周报自动生成

**触发**：每周日20:00
**动作**：自动生成周报

```
流程：
每周日20:00
→ 汇总本周数据
→ 生成周报模板
→ 填入数据
→ 发送到飞书/邮箱
```

## 三、Zapier 实操教程

### 注册与设置

1. 访问 zapier.com
2. 注册账号（免费版100任务/月）
3. 连接你的应用（Notion/Gmail/Slack等）

### 创建第一个自动化

**示例：Gmail新邮件 → Notion记录**

```
Step 1：创建新Zap
- 点击 "Create Zap"

Step 2：设置触发器
- 选择 "Gmail"
- 事件选择 "New Email"
- 连接你的Gmail账号
- 设置筛选条件（如：包含"咨询"）

Step 3：设置动作
- 选择 "Notion"
- 事件选择 "Create Database Item"
- 连接你的Notion账号
- 映射字段：
  - 邮件标题 → 客户名称
  - 邮件内容 → 需求描述
  - 发件人 → 联系方式

Step 4：测试
- 点击 "Test & Review"
- 确认数据正确

Step 5：开启
- 点击 "Turn on Zap"
```

### 常用Zap模板

| 模板 | 触发 | 动作 |
|------|------|------|
| 新邮件→Slack通知 | Gmail新邮件 | Slack发送消息 |
| 新文件→备份 | Google Drive新文件 | Dropbox备份 |
| 新订单→发票 | Stripe新订单 | QuickBooks创建发票 |
| 新表单→CRM | Typeform新回复 | HubSpot创建联系人 |

## 四、Make 实操教程

### 优势

- 免费额度更多（1000操作/月）
- 可视化流程设计
- 支持更复杂的逻辑

### 创建流程

```
Step 1：创建新场景
- 点击 "Create a new scenario"

Step 2：添加模块
- 搜索并选择触发器应用
- 配置触发条件

Step 3：添加动作
- 点击 "+" 添加新模块
- 选择动作应用
- 配置执行内容

Step 4：连接模块
- 用线连接各模块
- 设置数据映射

Step 5：测试运行
- 点击 "Run once"
- 查看执行结果

Step 6：设置定时
- 设置执行频率（每小时/每天/每周）
- 开启场景
```

## 五、自动化最佳实践

### 设计原则

| 原则 | 说明 |
|------|------|
| 先手动再自动 | 先手动跑通流程，再自动化 |
| 从简单开始 | 先做2-3步的简单流程 |
| 加错误处理 | 设置失败通知和重试 |
| 定期检查 | 每周检查自动化是否正常运行 |
| 记录文档 | 记录每个自动化的用途和设置 |

### 常见坑

| 问题 | 解决方案 |
|------|---------|
| 触发器不触发 | 检查连接是否过期，重新授权 |
| 数据格式错误 | 添加格式转换步骤 |
| 重复执行 | 设置去重逻辑 |
| 免费额度不够 | 合并多个自动化，减少触发频率 |
| 调试困难 | 用测试数据，开启日志记录 |

### 自动化优先级

| 优先级 | 场景 | 原因 |
|--------|------|------|
| P0 | 成交记录 | 直接影响收入 |
| P0 | 客户跟进 | 避免遗漏客户 |
| P1 | 内容提醒 | 保持更新频率 |
| P1 | 数据汇总 | 节省统计时间 |
| P2 | 素材收集 | 提高效率 |
| P2 | 竞品监控 | 辅助决策 |
